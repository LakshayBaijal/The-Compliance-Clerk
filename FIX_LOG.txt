PROBLEM IDENTIFIED:
- Extraction success rate dropped from 96.44% to 0%
- All 225 pages marked as FAILED with confidence 0.0
- CSV export was also failing with field mismatch errors

ROOT CAUSE ANALYSIS:
The early-exit logic in main.py (lines 143-160) was terminating image-only 
pages BEFORE they could be processed:

    if (not text or not text.strip()):
        if has_images:
            if not force_llm_for_images:
                # EXIT HERE with confidence 0.0
                continue

This meant that even though ImageOnlyExtractor could extract from filenames,
the code never reached it because it exited early with "failed" status.

   │         → Can now use ImageOnlyExtractor fallback            │
   └─────────────────────────────────────────────────────────────┘

   Changed:

   - Restructured to skip only pages with NO text AND NO images
